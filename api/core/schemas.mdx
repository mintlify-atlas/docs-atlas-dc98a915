# Schemas

All Mega Brain state and artifacts use JSON Schema validation for data integrity.

## Schema Index

| Schema | State File | Purpose |
|--------|------------|----------|
| `chunks-state.schema.json` | `CHUNKS-STATE.json` | Semantic chunks with embeddings |
| `canonical-map.schema.json` | `CANONICAL-MAP.json` | Entity canonicalization mappings |
| `insights-state.schema.json` | `INSIGHTS-STATE.json` | Extracted insights by priority |
| `narratives-state.schema.json` | `NARRATIVES-STATE.json` | Synthesized narratives |
| `file-registry.schema.json` | `file-registry.json` | Processed file tracking |
| `decisions-registry.schema.json` | `decisions-registry.json` | Council decisions and precedents |

**Location:** `core/schemas/`

## chunks-state.schema.json

Validates chunk state with embeddings and metadata.

**Schema Version:** 1.0.0

### Structure

```json
{
  "metadata": {
    "version": 1,
    "created_at": "2026-03-05T12:00:00Z",
    "updated_at": "2026-03-05T14:30:00Z",
    "total_sources": 15,
    "total_chunks": 342
  },
  "chunks_by_source": {
    "CG001": {
      "source_id": "CG001",
      "source_name": "Cole Gordon",
      "source_file": "inbox/COLE-GORDON/PODCASTS/farm-system.txt",
      "source_hash": "sha256:...",
      "chunk_count": 23,
      "processed_at": "2026-03-05T12:00:00Z",
      "chunks": [
        {
          "chunk_id": "CG001-001",
          "content": "The farm system is...",
          "word_count": 847,
          "embedding": [0.123, -0.456, ...],  // 1024-dim vector
          "persons_mentioned": ["Cole Gordon"],
          "roles_mentioned": ["CLOSER", "BDR"],
          "themes": ["02-PROCESSO-VENDAS"],
          "priority": "HIGH",
          "metadata": {
            "chunk_index": 0,
            "start_char": 0,
            "end_char": 5234
          }
        }
      ]
    }
  },
  "change_log": [
    {
      "timestamp": "2026-03-05T12:00:00Z",
      "action": "source_added",
      "source_id": "CG001",
      "chunk_count": 23
    }
  ]
}
```

### Field Definitions

<ResponseField name="metadata" type="object" required>
  Schema metadata and statistics
  
  <Expandable title="Properties">
    <ResponseField name="version" type="integer" required>
      Schema version (increments on changes)
    </ResponseField>
    
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 timestamp of last update
    </ResponseField>
    
    <ResponseField name="total_sources" type="integer" required>
      Count of unique sources
    </ResponseField>
    
    <ResponseField name="total_chunks" type="integer" required>
      Count of all chunks across sources
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="chunks_by_source" type="object" required>
  Dictionary mapping source_id to chunk data
  
  <Expandable title="Source Object">
    <ResponseField name="source_id" type="string" required>
      Unique source identifier (e.g., "CG001")
    </ResponseField>
    
    <ResponseField name="source_name" type="string" required>
      Human-readable source name
    </ResponseField>
    
    <ResponseField name="source_file" type="string" required>
      Original file path
    </ResponseField>
    
    <ResponseField name="source_hash" type="string" required>
      SHA-256 hash of source content
    </ResponseField>
    
    <ResponseField name="chunk_count" type="integer" required>
      Number of chunks for this source
    </ResponseField>
    
    <ResponseField name="chunks" type="array" required>
      Array of chunk objects
      
      <Expandable title="Chunk Object">
        <ResponseField name="chunk_id" type="string" required>
          Unique chunk ID: `{source_id}-{NNN}`
        </ResponseField>
        
        <ResponseField name="content" type="string" required>
          Chunk text content (500-1500 words)
        </ResponseField>
        
        <ResponseField name="word_count" type="integer" required>
          Word count of content
        </ResponseField>
        
        <ResponseField name="embedding" type="array">
          1024-dimensional embedding vector (Voyage)
        </ResponseField>
        
        <ResponseField name="persons_mentioned" type="array">
          List of canonical person names
        </ResponseField>
        
        <ResponseField name="roles_mentioned" type="array">
          List of canonical role names
        </ResponseField>
        
        <ResponseField name="themes" type="array">
          List of theme codes (e.g., ["02-PROCESSO-VENDAS"])
        </ResponseField>
        
        <ResponseField name="priority" type="string">
          Priority level: `HIGH`, `MEDIUM`, `LOW`
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="change_log" type="array" required>
  Audit trail of all changes to this state file
</ResponseField>

### Validation

```python
import json
import jsonschema

# Load schema
with open('core/schemas/chunks-state.schema.json') as f:
    schema = json.load(f)

# Load data
with open('processing/chunks/CHUNKS-STATE.json') as f:
    data = json.load(f)

# Validate
jsonschema.validate(data, schema)
```

## ID System

### Source IDs

**Format:** `PREFIX` + `NNN`

**Examples:** `CG001`, `JL003`, `HR005`

**Registered Prefixes:**

| Prefix | Person/Channel | Company |
|--------|----------------|--------|
| JL | Jordan Lee | AI Business |

| MT | Max Tornow | Max Tornow Podcast |
| HR | Alex Hormozi | - |
| CG | Cole Gordon | - |
| SS | Sam Oven | Setterlun University |
| JM | Jeremy Miner | 7th Level |

### Chunk IDs

**Format:** `{SOURCE_ID}-{NNN}`

**Examples:** `CG001-001`, `JL003-015`

### Decision IDs

**Format:** `YYYYMMDDHHMMSS-{ORIGIN}-{DEST}`

**Example:** `20260305130249-CRO-CFO`

### Precedent IDs

**Format:** `PREC-YYYY-NNN`

**Example:** `PREC-2026-001`

## Validation Tools

### Python

```python
import json
import jsonschema

def validate_state_file(state_file, schema_file):
    with open(schema_file) as f:
        schema = json.load(f)
    with open(state_file) as f:
        data = json.load(f)
    
    try:
        jsonschema.validate(data, schema)
        return True, "Valid"
    except jsonschema.ValidationError as e:
        return False, str(e)
```

### CLI

```bash
# Validate all state files
python3 core/intelligence/validate_json_integrity.py

# Validate single file
python3 -m jsonschema -i CHUNKS-STATE.json core/schemas/chunks-state.schema.json
```

## Schema Evolution

### Version Increment Rules

1. **Never delete fields** - Mark as deprecated
2. **Always validate before save** - Use jsonschema
3. **Increment version** on each schema change
4. **Maintain change_log** for auditability

### Migration

When schema changes:

1. Create migration script: `scripts/migrate_v{N}_to_v{N+1}.py`
2. Update schema file with new version
3. Run migration on all state files
4. Validate with new schema
