Which one of those is closest for you?"

Then SHUT UP. Let them tell you the real objection.

### Response Structure

Every mind clone response includes:

1. **Direct Answer** - In the expert's voice
2. **Framework Application** - Named methodologies used
3. **Sources** - Chunk_ids linking to original material
4. **Confidence Score** - 0.00-1.00 based on evidence weight
5. **Related Concepts** - Additional relevant frameworks

<Tip>
Mind clones only use knowledge with weight ≥ 0.70 to ensure high-confidence responses.
</Tip>

## Agent Auto-Enrichment

Agents are automatically updated as you process materials:

<Accordion title="How Auto-Enrichment Works">

### Phase 7: Agent Enrichment (Pipeline)

When processing completes:

```
1. Extract themes and frameworks from insights
2. Map themes to relevant agents using THEME_TO_AGENTS:

   "02-PROCESSO-VENDAS" → [closer, SDS, LNS]
   "04-COMISSIONAMENTO" → [sales-manager, CRO, CFO]
   "07-PRICING" → [CRO, CFO, closer]

3. For each mapped agent:
   - Load existing MEMORY-{agent}.md
   - Append new frameworks
   - Add source reference
   - Update last_modified timestamp

4. Verify coverage with Agent Coverage Check
```

### Framework-Specific Routing

Certain frameworks trigger specific agents:

```yaml
"3 Audience Buckets": [closer, SDS, LNS]
"NEPQ Framework": [closer, SDS]
"Email Hammer": [LNS]
"Farm System": [sales-manager, CRO]
"Christmas Tree Structure": [sales-manager, CRO]
```

### Coverage Verification

Pipeline Phase 8.6 verifies:
```
☐ AGENT COVERAGE CHECK (Mandatory):

1. Identify all themes in SOURCE_ID
2. For each theme, lookup THEME_TO_AGENTS
3. For each framework, lookup FRAMEWORK_TO_AGENTS
4. List ALL agents that SHOULD receive content
5. Verify grep -l "$SOURCE_ID" in each MEMORY-{agent}.md
6. IF any agent missing SOURCE_ID:
   → LOG ERROR: "Agent Coverage Failed"
   → List expected vs. actual agents
   → EXIT with status: AGENT_COVERAGE_FAILED
```

</Accordion>

## Agent Creation Workflow

### Automatic Creation (Threshold-Based)

Agents are auto-created when roles reach thresholds:

<Steps>

<Step title="Role Detection">
As materials are processed, mentions of roles are counted:

```
CLOSER: 21.6 weighted mentions → TIER 1 (Established)
APPOINTMENT-SETTER: 12.3 mentions → TIER 1 (Create agent)
COPYWRITER: 7.8 mentions → TIER 2 (Emerging, monitor)
PODCAST-HOST: 2.1 mentions → TIER 3 (Emergent, observe)
```
</Step>

<Step title="Threshold Check">
**TIER 1 (Established):** weighted_score ≥ 10, sources ≥ 2
  → **Action:** CREATE AGENT

**TIER 2 (Emerging):** weighted_score ≥ 5, sources ≥ 1
  → **Action:** Monitor growth

**TIER 3 (Emergent):** weighted_score ≥ 3, sources ≥ 1  
  → **Action:** Observe
</Step>

<Step title="Auto-Creation Trigger">
When TIER 1 reached:
```
python scripts/agent_creator.py --check
```

Creates:
- `agents/cargo/SALES/AGENT-APPOINTMENT-SETTER.md`
- `agents/cargo/SALES/MEMORY-APPOINTMENT-SETTER.md`
- Corresponding SUA-EMPRESA files (org roles)
</Step>

</Steps>

### Manual Creation

Create agents manually:

```bash
/create-agent APPOINTMENT-SETTER --category SALES --sub-of SALES-MANAGER
```

<Warning>
Manual creation bypasses threshold checks. Use for special cases only.
</Warning>

## Troubleshooting

<Accordion title="Common Issues">

**Issue: Agent says "I don't have knowledge about that"**

Solutions:
1. Check if agent has been enriched with relevant sources:
   ```bash
   /agents --knowledge "[expert-name]"
   ```
2. Process more materials on that topic
3. Use a different agent (e.g., mind clone vs. cargo)

**Issue: Response lacks sources/chunk_ids**

Causes:
- Agent memory not properly updated
- Pipeline didn't complete Phase 7
- Sources missing from MEMORY-{agent}.md

Fix:
```bash
# Reprocess materials to update agent memories
/process-jarvis inbox/[person]/[file].txt
```

**Issue: Agent is outdated (>3 days)**

```bash
# Check which agents need updates
/agents --outdated

# Process new materials to refresh
/process-jarvis [new-materials]
```

**Issue: Want to create agent but below threshold**

```bash
# Check current mentions
/agents

# Either:
# 1. Process more materials mentioning that role
# 2. Force creation (not recommended):
/create-agent [ROLE] --force
```

</Accordion>
